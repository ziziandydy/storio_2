## ADDED Requirements

### Requirement: Problem & Vision Narrative
The landing page SHALL explicitly address the problem of fragmented media tracking and present Storio's vision as a "Digital Pensieve".

#### Scenario: Problem Section
- **WHEN** the user scrolls past the Hero section
- **THEN** they SHALL see a "Problem" section identifying the pain points of scattered platforms like Letterboxd, IMDb, and Goodreads.

#### Scenario: Vision Section
- **WHEN** the user views the "Vision" section
- **THEN** they SHALL read about the "Pensieve" philosophy—a quiet, private space for curating personal relationships with stories.

### Requirement: Four Core Features Showcase
The page SHALL showcase exactly 4 core features of Storio 2.

#### Scenario: Feature Grid
- **WHEN** the user views the Features section
- **THEN** they SHALL see the following 4 features:
  1. **Memory Timeline**: Separate cards for every rewatch/reread.
  2. **Visual Templates**: Cinema Ticket, 3D Shelf, and Desk flat-lay sharing.
  3. **Bottom-Focused Interface**: Single-hand optimized search and navigation.
  4. **Dynamic Stats Dashboard**: Customizable insights into personal collection trends.

### Requirement: Builder's Story (About Section)
The page SHALL include an "About" section describing the developer's journey.

#### Scenario: Reading the About Section
- **WHEN** the user reaches the About section
- **THEN** they SHALL read the story of a **Data PM transitioning to Build PM**.
- **AND** it MUST emphasize that the project was built using **Pure Vibe Coding + AI Tools** starting from zero coding ability.

### Requirement: Multi-Language Support
The landing page SHALL support switching between English and Traditional Chinese.

#### Scenario: Language Toggling
- **WHEN** the user clicks the language toggle button in the header
- **THEN** all UI text (Hero, Features, About, etc.) SHALL immediately update to the selected language.
- **AND** the preference SHALL be saved for future visits.

### Requirement: Refined Footer
The footer SHALL contain professional copyright information and personal links.

#### Scenario: Footer Interaction
