# PRD: Collection Views & Smart Switcher (多重館藏視圖)

## 概述
為使用者的館藏頁面 (Folio) 實作多重檢視系統，允許使用者以「列表 (List)」、「行事曆 (Calendar)」和「畫廊 (Gallery)」模式瀏覽收藏。切換機制將是一個位於計數器旁的單一按鈕，採用循環切換設計，並能記憶使用者的偏好設定。

## 目標
- 提供量身打造的瀏覽體驗（效率列表、回顧行事曆、沉浸畫廊）。
- 透過單一循環按鈕保持介面簡潔，不佔用 Header 空間。
- 記憶使用者最後選擇的視圖模式。

## 品質門檻 (Quality Gates)

每個使用者故事 (User Story) 都必須通過以下指令：
- `npm run lint` - 程式碼風格檢查
- `npm run typecheck` - 型別檢查

對於涉及 UI 的故事，還必須：
- 使用 `dev-browser` 技能在模擬瀏覽器中驗證佈局與互動。

## 使用者故事 (User Stories)

### US-13A: 智慧視圖切換器 (Smart View Switcher)
**描述:** 作為一名使用者，我希望在館藏計數器旁有一個切換按鈕，讓我能循環切換不同的瀏覽模式，而不增加介面的雜亂感。

**驗收標準:**
- [ ] 在館藏頁面 Header 的 `[n]/[limit]` 計數器右側放置一個圖示按鈕。
- [ ] 點擊按鈕應依序循環切換：`列表視圖` -> `行事曆視圖` -> `畫廊視圖` -> `列表視圖`。
- [ ] 按鈕圖示需隨**當前**模式變更（例如：列表圖示、日曆圖示、圖片圖示）。
- [ ] 切換時顯示微動畫（如翻轉或淡入淡出）。
- [ ] 切換瞬間顯示短暫的 Toast 提示（例如：「已切換至行事曆模式」）。
- [ ] 將選擇的視圖模式儲存至 `localStorage`，重新整理後保持該模式。

### US-13B: 行事曆視圖 (Calendar View)
**描述:** 作為一名使用者，我希望按加入日期查看我的收藏，以便回顧我的閱讀/觀影旅程。

**驗收標準:**
- [ ] 渲染一個月曆網格。
- [ ] **單一項目日期**: 在日期格子中顯示該項目的封面/海報縮圖。
- [ ] **多項目日期**: 顯示最新項目的封面，並加上「堆疊卡片」視覺效果（底部有層次陰影）以暗示有多個項目。**不顯示數字徽章**。
- [ ] 點擊日期格子應展開一個小列表或 Overlay，顯示該日期的所有項目。
- [ ] 沒有收藏的日期僅顯示日期數字。

### US-13C: 畫廊視圖 (Gallery View / Carousel)
**描述:** 作為一名使用者，我希望以沉浸式的橫向捲動方式瀏覽我的收藏，專注欣賞作品的視覺藝術。

**驗收標準:**
- [ ] 實作橫向捲動 / 旋轉木馬 (Carousel) 佈局（例如使用 `embla-carousel` 或類似輕量庫）。
- [ ] 每張卡片應佔據大部分螢幕寬度（例如 80%），強調海報視覺。
- [ ] 在卡片上顯示極簡資訊（標題、評分）。
- [ ] 背景可選用當前海報的模糊或變暗版本，增加沉浸感。

### US-13D: 視圖狀態整合 (View State Integration)
**描述:** 作為一名使用者，我希望在切換視圖時能有流暢的轉場體驗。

**驗收標準:**
- [ ] 重構 `/collection/page.tsx`，根據狀態條件渲染對應的視圖元件 (`ListView`, `CalendarView`, `GalleryView`)。
- [ ] 確保「搜尋/篩選」列在不同視圖下的行為合理（預設：列表模式可見，畫廊模式可能自動隱藏）。
- [ ] 確保切換視圖時有平滑的轉場動畫（如 Cross-fade）。

## 功能需求
- FR-1: 新使用者的預設視圖為 **列表視圖 (List View)**。
- FR-2: 行事曆視圖必須支援月份切換。
- FR-3: 畫廊視圖在手機上必須支援觸控滑動手勢。

## 非目標 (Non-Goals)
- 畫廊視圖中的拖放排序功能。
- 行事曆視圖中的複雜篩選（篩選邏輯由底層資料處理，行事曆僅負責顯示）。

## 技術考量
- 畫廊視圖建議使用輕量級 Carousel 套件，或 CSS Scroll Snap。
- 行事曆邏輯可客製化或深度修改 `react-day-picker` 以支援「格子內內容」。
- 確保圖片經過優化 (Next.js Image)，避免在畫廊/行事曆模式下造成卡頓。

## 成功指標
- 使用者使用切換按鈕的頻率。
- 切換視圖時無佈局偏移 (CLS)。
